# Club Activity Management System - Docs

本リポジトリは、  
**部活動向け 業務支援システム** の設計ドキュメントを管理するためのものです。

本システムは、部活動における欠席連絡や試合結果報告を  
口頭やチャットによる「連絡」ではなく、  
**申請 → 承認 → 証跡** として管理することを目的としています。

---

##  システム概要
本システムでは、以下の課題解決を目指します。

- 欠席・遅刻等の連絡が属人化し、後から確認できない
- 顧問が把握していたかどうかが曖昧になりやすい
- 試合結果が非公式な情報のまま残る
- 判断や責任の所在が不明確

これらに対し、  
**業務システムの考え方（申請・承認・証跡）** を部活動に適用します。

---

##  提供機能（MVP）
- **欠席系の申請・承認管理**
  - 欠席／遅刻／早退／不参加
  - 申請 → 承認 → 履歴保存
- **試合結果報告の申請・承認管理**
  - 承認後のデータのみを公式記録として扱う
- **練習スケジュール共有**
  - 監督・コーチが管理、生徒は閲覧のみ

---

##  想定ロール
| ロール | 概要 |
|---|---|
| 生徒 | 申請・報告の作成、結果確認、スケジュール閲覧 |
| 監督・コーチ（顧問） | 申請・報告の承認、練習スケジュール管理 |
| 管理者 | 部活・メンバー管理、ログ閲覧 |

##  ドキュメント構成
本リポジトリは、以下の構成でドキュメントを管理しています。

```text
docs/
├── 00_overview/        # システム概要
├── 01_requirements/    # 要件定義
├── 02_ui/              # 画面一覧・画面遷移
├── 03_workflow/        # 状態遷移・操作ルール
├── 04_domain/          # ER図・DB設計方針
├── 05_api/             # API仕様・エラー設計
├── 06_infra/           # 構成図・デプロイ設計
└── diagrams/           # 各種図（画像）
```

- Backend: Spring Boot（Java）
- DB: PostgreSQL
- Auth: Spring Security
